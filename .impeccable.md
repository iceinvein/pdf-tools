# Impeccable Design Context

## Users

People on the web who need to manipulate PDFs quickly — view, delete pages, extract, merge, reorder. They're typically at a desk during work hours, often in a browser tab alongside other tools. They chose this over downloading desktop software or creating an account somewhere. They want to get in, do the thing, and get out.

**Key insight**: These users are mid-task. They have a PDF problem to solve *right now*. The interface should feel like it was already waiting for them.

## Brand Personality

**Three words**: Sharp, capable, effortless

**Emotional goal**: "This just works" — the feeling of a well-made hand tool. No learning curve, no friction, no delight-for-delight's-sake. Competence is the delight.

**Voice**: Direct, minimal, confident. Labels say what they do. No clever copy, no onboarding wizards, no "Welcome to PDF Tools!" Just the tool.

## Aesthetic Direction

**Tone**: Refined utilitarian — the aesthetic of a well-designed workshop tool. Not cold or clinical, but purposeful. Every surface earns its place.

**Reference**: Notion's spatial confidence — generous whitespace, approachable density, type-driven hierarchy. Not Notion's feature complexity or its playful emoji culture. Specifically: how Notion makes a dense workspace feel calm through spacing and restraint.

**Anti-references**:
- Generic SaaS dashboards with card grids and metric heroes
- Dark-mode-with-neon / glassmorphism / AI landing page aesthetic
- Overly playful or illustrated tool UIs (Canva, smallpdf's illustration style)
- Cramped toolbar-heavy PDF viewers (Adobe Acrobat, browser PDF viewers)

**Theme**: Light mode primary. This is a daytime desk tool. The user is already staring at documents — the UI should feel like clean paper, not a screen.

**Color direction**: Warm neutrals tinted toward the brand hue. A single, sharp accent color used sparingly — for active states, CTAs, and the one thing that needs attention right now. No rainbow, no gradients.

## Design Principles

1. **Clarity over decoration** — Every element communicates something. If it doesn't help the user manipulate their PDF, it doesn't belong.
2. **Earned density** — Start sparse (drag-and-drop), get denser as the user goes deeper (toolbar, page navigation). Complexity reveals itself through use, not upfront.
3. **Confident whitespace** — Generous spacing signals quality and competence. A PDF tool with breathing room says "we thought about this."
4. **Real identity, not defaults** — Custom font pairing, considered palette, proper icon system. Not a Tailwind template with shadcn defaults.
5. **Tools, not toys** — Every interaction serves a function. Animations are fast state transitions, not performances. Respect the user's time above all.

## Technical Constraints

- React 19 + TypeScript + Vite 7
- Tailwind CSS v4 with OKLCH color system (already in place)
- shadcn/ui components as a base (customized, not stock)
- Lucide icons
- WCAG AA compliance
- Canvas-based PDF rendering (pdfjs-dist)
- No server — all operations are client-side
